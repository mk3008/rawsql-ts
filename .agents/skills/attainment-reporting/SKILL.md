# Attainment Reporting

Use this skill when a rawsql-ts developer task is ready to be summarized in a PR or review report.

## Use It For
- Mapping each acceptance item to `done`, `partial`, or `not done`.
- Capturing evidence for each status.
- Explaining what was better than manual work and what remained insufficient.

## Workflow
1. List the acceptance items from the plan.
2. Map each item to a status.
3. Add the evidence that justifies the status.
4. Call out what was better than manual work.
5. Call out what was still insufficient and whether follow-up is needed.

## Output Shape
- Acceptance items status
- Outcome
- Why it matters
- Evidence
- Attainment level
- Follow-up

## Constraints
- Keep the report explicit and itemized.
- Do not hide partial completion behind a summary sentence.
