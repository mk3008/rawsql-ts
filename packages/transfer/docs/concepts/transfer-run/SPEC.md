# Transfer Run Concept Spec

## Position

この文書は、`@rawsql-ts/transfer` における Transfer Run の Concept Spec である。

Transfer Run は、`Transfer Execution` の引数を実行引数記録兼プロセスヘッダーとして記録し、その実行に対する進捗状態を管理する概念である。

この文書は、Transfer Run の意味、責務、非責務、不変条件を固定する。実装手順、SQL、DDL、転送処理の具体ロジックは定義しない。

## Definition

- Transfer Run とは、`Transfer Execution` が生成する実行引数記録兼プロセスヘッダーである。
- Transfer Run は、`Transfer Execution` の引数を記録する。
- Transfer Run は、実行開始時の引数と、実行全体のライフサイクル状態を保持する。
- Transfer Run は、必ず1つの `Transfer Setting` を対象にする。
- `Transfer Setting` が分かれば、その転送設定に紐づく `Destination` 群を特定できる。
- Transfer Run は、`Transfer Execution` に従属する。
- Transfer Run は、`Transfer Execution` が生成する実行引数記録兼プロセスヘッダーであり、事前に積まれるキューではない。

## Non-responsibilities

- Transfer Run は転送対象行を直接管理しない。
- Transfer Run は、どの行を転送すべきかを管理しない。
- 転送対象かどうかは、`Work Item` の文脈で判断する。
- Transfer Run は `Destination` の定義を持たない。
- Transfer Run は転送元 SQL や転送先 SQL を持たない。
- Transfer Run は、転送先行の由来追跡を管理しない。
- Transfer Run は、赤伝時に参照すべき現在有効な黒を管理しない。
- Transfer Run は、変更検知履歴の登録方法を定義しない。
- Transfer Run は、転送対象行がどのように固定化されるかの具体手順を定義しない。
- Transfer Run は、スケジューリングそのものを管理しない。

## Responsibilities

- Transfer Run は、どの `Transfer Setting` に対する実行かを示す。
- Transfer Run は、実行開始時の任意引数を保持する。
- Transfer Run は、実行全体のライフサイクル状態を示す。
- Transfer Run は、`Transfer Execution` のプロセスヘッダーとして、監査やデバッグで参照できる実行文脈を提供する。
- Transfer Run は、後続の `Lineage` や `Active Black` から参照できる実行単位を提供する。
- Transfer Run は、日次、月次、手動、再実行などの実行契機の由来や分類を記録してよい。
- Transfer Run は、必要に応じて転送元スナップショットを取った時点を説明できる情報を保持してよい。
- Transfer Run は、必要に応じて実行者、開始日時、終了日時、エラー情報などを保持してよい。

## Invariants

- Transfer Run は、必ず1つの `Transfer Setting` を対象にする。
- Transfer Run は、`Transfer Execution` によって生成される。
- Transfer Run は、`Transfer Execution` に従属する。
- Transfer Run は、実行開始時の引数を後から追跡できる形で保持する。
- Transfer Run の任意引数は、`Transfer Setting` の基礎 SQL で利用される可能性があるが、その解釈は Transfer Run 単独では決まらない。
- Transfer Run は、転送対象行の正本ではない。
- Transfer Run は、転送対象行を受け付けない。
- Transfer Run は、`Lineage` や `Active Black` から参照可能である必要がある。
- Transfer Run が転送元スナップショットの時点を保持する場合、その情報は `Lineage` から参照される実行文脈として扱う。
- Transfer Run の状態は、実行全体のライフサイクル状態を示すものであり、個別の実行試行、`Destination` ごとの成否、キーごとの処理結果を表すものではない。

## Why

### Transfer Run Targets One Transfer Setting

Transfer Run は、`Transfer Execution` が生成する実行引数記録兼プロセスヘッダーである。

Transfer Run は、必ず1つの `Transfer Setting` を対象にする。

`Transfer Setting` が分かれば、その `Transfer Setting` に紐づく `Destination` 群を特定できる。

Transfer Run は、`Transfer Execution` がどの `Transfer Setting` を実行対象にしたかを示す。

### Run Arguments Are Run Conditions

Transfer Run は、`Transfer Execution` の実行開始時任意引数を持ってよい。

任意引数の例:

- 処理日
- 対象年月
- 支店
- 顧客
- 実行者
- 再実行理由
- 手動実行フラグ

これらの引数は、`Transfer Setting` の基礎 SQL で条件値として利用される可能性がある。

ただし、Transfer Run はそれらの引数をどのように SQL へ反映するかを定義しない。
任意検索条件の具体的な扱いは、`Transfer Setting` の基礎 SQL と [SSSQL](../../../../../docs/guide/sssql-overview.md) に従う。

### Run Status Is Run Lifecycle State

Transfer Run は、実行に対する進捗状態を持ってよい。

状態の例:

```yaml
transfer_run_status:
  - created
  - running
  - succeeded
  - failed
  - cancelled
```

この状態は、実行全体の状態を示す。
個別の実行試行、`Destination` ごとの成否、キーごとの処理結果を表すものではない。

状態値の最終的な集合は、後続の実装 Issue で決めてよい。
ただし、Transfer Run の状態は「実行全体の進捗」を表す、という意味を変えてはならない。

### Transfer Run Is Generated by Execution

Transfer Run は、`Transfer Execution` が生成するプロセスヘッダーである。

`Transfer Execution` が Transfer Run を取り出して実行するのではなく、`Transfer Execution` が実行引数を Transfer Run として記録する。

### Transfer Run Does Not Own Target Rows

Transfer Run は、転送実行の条件を管理する。

Transfer Run は、転送対象行を受け付けるものではない。
どの行を転送すべきかは、`Work Item` の文脈で判断する。

Transfer Run は、転送対象行の正本ではない。

### Transfer Run Is Referenced by Later Records

Transfer Run は、後続の `Lineage` や `Active Black` から参照される実行単位である。

`Lineage` は、どの転送元キーが、どの転送先キーへ、どの実行で転送されたかを記録する。
`Lineage` が参照するスナップショットの時点は、Transfer Run の実行文脈として説明される。

Transfer Run は、`Lineage` が参照する実行文脈である。

`Active Black` は、現在有効な黒伝がどの実行文脈で有効になったかを説明するために、Transfer Run を参照してよい。

## Usage Notes

- Transfer Run は、`Transfer Execution` の実行引数記録兼プロセスヘッダーとして読む。
- Transfer Run は、日次、月次、手動、再実行などの実行契機の由来や分類を記録してよい。ただし、スケジューリングそのものは管理しない。
- Transfer Run は、実行条件を保持する。転送対象行そのものは保持しない。
- Transfer Run の任意引数は、基礎 SQL の SSSQL に渡される値として利用される可能性がある。
- Transfer Run は、必要に応じて転送元スナップショットを取った時点を説明する。`Lineage` はその実行文脈を参照する。
- Transfer Run の状態は、実行全体のライフサイクル状態として読む。個別実行試行や `Destination` ごとの詳細結果は別責務として扱う。

### Out of Scope

転送処理は、作業対象行の特定と実際の転送に分けて設計される可能性がある。

ただし、この文書では、その具体的な処理順序、トランザクション境界、ワークアイテムの確定方法は定義しない。

それらは、別の Concept Spec、feature spec、または後続 Issue で扱う。
